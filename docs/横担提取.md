# 横担位置提取逻辑

本文档用于说明：在当前模型没有显式分割出横担类别的情况下，如何借助已经分割出的 **杆塔、导线、绝缘子** 来推断横担位置，并输出每个杆塔上横担的最左侧和最右侧位置。

当前四分类语义约定：

```text
0：背景 / 地面
1：杆塔
2：导线
3：绝缘子
```

当前没有横担类别，所以横担不能直接通过 `classification == 某个类别` 提取。  
横担需要通过几何关系推断：

```text
横担通常位于杆塔上部；
横担两侧通常连接绝缘子；
绝缘子附近通常连接导线；
横担方向通常近似垂直于线路走向。
```

---

## 1. 目标输出

最终希望对每个杆塔输出横担的左右端位置：

```json
{
  "input": "xxx_clean.las",
  "coordinate_system": "las_global_xyz",
  "crossarms": [
    {
      "tower_id": 1,
      "tower_name": "杆塔1",
      "crossarm_id": 1,
      "crossarm_name": "杆塔1_横担1",
      "left_point_xyz": [526000.12, 3514000.35, 86.20],
      "right_point_xyz": [526008.64, 3514000.41, 86.35],
      "center_xyz": [526004.38, 3514000.38, 86.28],
      "height_z": 86.28,
      "direction": {
        "along_axis_xy": [0.02, 1.00],
        "side_axis_xy": [1.00, -0.02]
      },
      "support": {
        "insulator_points": 320,
        "line_points": 1800,
        "tower_points": 4500
      }
    }
  ]
}
```

字段说明：

- `left_point_xyz`：横担左端点，全局 LAS 坐标。
- `right_point_xyz`：横担右端点，全局 LAS 坐标。
- `center_xyz`：左右端点的中点。
- `height_z`：横担所在高度。
- `along_axis_xy`：线路走向方向。
- `side_axis_xy`：横担左右方向，通常垂直于线路走向。
- `support`：用于推断该横担的点数依据，方便检查可靠性。

注意：所有坐标都应该使用 LAS 全局坐标，也就是 CloudCompare 右下角显示的坐标。

---

## 2. 总体思路

横担没有独立分割类别，所以不能直接聚类横担点。  
推荐使用“关系推断”的方式：

```text
读取 LAS
  -> 提取杆塔点、导线点、绝缘子点
  -> 对杆塔点聚类，得到每个杆塔实例
  -> 对绝缘子点聚类，得到每个绝缘子实例
  -> 将绝缘子实例绑定到最近杆塔
  -> 根据导线点估计线路走向 along 方向
  -> 横担方向 side = along 在 XY 平面内的垂直方向
  -> 在每个杆塔附近，按绝缘子高度分层
  -> 每一层绝缘子对应一个横担候选
  -> 在该高度层内，沿 side 方向取最左和最右位置
  -> 输出横担左右端点
```

核心不是找“横担点”，而是找：

```text
绝缘子和导线连接杆塔的位置。
```

这些位置通常正好落在横担附近。

---

## 3. 第一步：读取点云类别

使用 `laspy` 读取 LAS：

```python
import laspy
import numpy as np

las = laspy.read(input_path)
coord = np.column_stack([las.x, las.y, las.z]).astype(np.float64)
cls = np.asarray(las.classification)
```

按类别拆分：

```python
tower_coord = coord[cls == 1]
line_coord = coord[cls == 2]
insulator_coord = coord[cls == 3]
```

如果输入 LAS 中含有画框脚本追加的 class `30/31` 可视化点，应忽略它们。

---

## 4. 第二步：提取杆塔实例

横担必须依附于杆塔，所以第一步需要得到每个物理杆塔。

可以复用后处理脚本里的杆塔聚类逻辑：

```text
1. 对 class == 1 的杆塔点做体素化。
2. 对体素做连通域聚类。
3. 删除点数太少、高度太低、形状异常的组件。
4. 合并同一物理杆塔的多个碎片。
5. 得到 physical_towers。
```

如果单独写横担提取脚本，简化版可以使用 DBSCAN：

```python
from sklearn.cluster import DBSCAN

cluster = DBSCAN(
    eps=1.5,
    min_samples=30,
    n_jobs=-1,
).fit(tower_coord[:, :2])

tower_labels = cluster.labels_
```

但是更推荐使用后处理脚本已经清理后的 LAS，因为里面的杆塔误分已经少一些。

每个杆塔实例需要保存：

```text
tower_id
tower_points
tower_center_xyz
tower_bbox_min_xyz
tower_bbox_max_xyz
tower_height
```

---

## 5. 第三步：提取绝缘子实例

由于横担通常与绝缘子相连，所以绝缘子是推断横担最重要的依据。

对 `classification == 3` 的点做实例聚类：

```python
from sklearn.cluster import DBSCAN

cluster = DBSCAN(
    eps=0.8,
    min_samples=15,
    n_jobs=-1,
).fit(insulator_coord)

insulator_labels = cluster.labels_
```

每个绝缘子实例保存：

```text
insulator_id
points
center_xyz
top_point_xyz
bottom_point_xyz
bbox_min_xyz
bbox_max_xyz
height
```

建议过滤：

```text
点数 < 30：删除
高度 < 0.2m：删除
距离杆塔太远：删除
距离导线太远：删除
```

---

## 6. 第四步：绝缘子绑定到杆塔

对每个绝缘子实例，找到最近的杆塔实例。

简单方法：

```text
计算绝缘子中心点到每个杆塔中心线或杆塔包围盒的 XY 距离；
选择距离最近的杆塔；
如果距离超过阈值，则认为该绝缘子不属于任何杆塔。
```

推荐阈值：

```text
max_insulator_to_tower_xy = 8.0 米
```

更稳的方法：

```text
将杆塔包围盒在 XY 方向外扩 6~8 米；
如果绝缘子中心或绝缘子点落在外扩区域内，则绑定到该杆塔。
```

绑定后，每个杆塔会有一组绝缘子：

```text
tower.insulators = [insulator1, insulator2, ...]
```

如果一个杆塔没有任何绝缘子，则无法可靠推断横担。

---

## 7. 第五步：估计线路走向 along 和横担方向 side

横担的左右方向需要有一个局部坐标系，否则“左”和“右”会不稳定。

### 局部坐标系定义

对每个杆塔，定义：

```text
Z 轴：高度方向，使用 LAS 全局 Z。
along 轴：线路走向方向。
side 轴：横担左右方向，垂直于 along。
```

在 XY 平面中：

```text
along_axis_xy = 线路方向单位向量
side_axis_xy = [-along_y, along_x]
```

其中：

```python
side_axis_xy = np.array([-along_axis_xy[1], along_axis_xy[0]])
```

### 如何估计 along 方向

方法一：使用该杆塔附近导线点做 PCA。

```text
1. 取杆塔附近一定范围内的导线点。
2. 只使用 XY 坐标。
3. PCA 第一主方向就是线路走向 along。
```

示例：

```python
near_line_xy = near_line_coord[:, :2]
mean_xy = near_line_xy.mean(axis=0)
centered = near_line_xy - mean_xy
_, _, vh = np.linalg.svd(centered, full_matrices=False)
along_axis_xy = vh[0]
along_axis_xy = along_axis_xy / np.linalg.norm(along_axis_xy)
side_axis_xy = np.array([-along_axis_xy[1], along_axis_xy[0]])
```

方法二：使用相邻杆塔中心连线。

```text
如果已经有杆塔1、杆塔2、杆塔3，则相邻杆塔中心连线可以作为线路走向。
```

对于中间杆塔：

```text
along = 后一个杆塔中心 - 前一个杆塔中心
```

对于端部杆塔：

```text
along = 相邻杆塔中心 - 当前杆塔中心
```

推荐优先级：

```text
优先使用相邻杆塔中心估计 along；
如果只有单塔或杆塔排序不可靠，再使用附近导线点 PCA。
```

---

## 8. 第六步：按高度分层找到横担候选

一座杆塔可能有多层横担。  
绝缘子通常挂在横担附近，所以可以通过绝缘子的中心高度进行分层。

对某个杆塔绑定到的绝缘子实例，取每个绝缘子的中心高度：

```python
z_values = [ins.center_xyz[2] for ins in tower.insulators]
```

按高度从高到低排序，然后根据高度间隔分层：

```text
如果两个绝缘子中心的 Z 差值小于 layer_z_gap，则认为在同一层横担；
如果 Z 差值大于 layer_z_gap，则分成不同横担层。
```

推荐参数：

```text
crossarm_layer_z_gap = 2.0 米
```

示例：

```text
第 1 层：Z ≈ 90m
第 2 层：Z ≈ 84m
第 3 层：Z ≈ 78m
```

每一层可以生成一个横担候选。

---

## 9. 第七步：计算横担最左和最右位置

对某一层横担，使用该层绝缘子和附近导线共同推断横担左右端。

### 坐标投影

设杆塔中心为：

```python
tower_center_xy = tower.center_xyz[:2]
```

对该层的候选点，例如绝缘子点和附近导线点，投影到局部 side 方向：

```python
local_side = (points[:, :2] - tower_center_xy[None, :]) @ side_axis_xy
```

`local_side` 越小，表示越靠左；  
`local_side` 越大，表示越靠右。

### 候选点来源

推荐候选点使用：

```text
该层绝缘子点
+ 该层附近的导线点
+ 杆塔上部附近的杆塔点，可选
```

高度范围：

```text
layer_center_z - z_margin <= point_z <= layer_center_z + z_margin
```

推荐：

```text
z_margin = 1.5 ~ 3.0 米
```

XY 范围：

```text
距离杆塔中心 XY 不超过 15 ~ 25 米
```

### 左右端点计算

为了避免单个离群点影响，不建议直接取最小值和最大值。  
推荐使用百分位数：

```python
left_side = np.percentile(local_side, 2)
right_side = np.percentile(local_side, 98)
```

对应的全局 XY：

```python
left_xy = tower_center_xy + left_side * side_axis_xy
right_xy = tower_center_xy + right_side * side_axis_xy
```

横担高度：

```python
crossarm_z = np.median(layer_points[:, 2])
```

最终左右点：

```python
left_point_xyz = [left_xy[0], left_xy[1], crossarm_z]
right_point_xyz = [right_xy[0], right_xy[1], crossarm_z]
```

中心点：

```python
center_xyz = (left_point_xyz + right_point_xyz) / 2
```

---

## 10. 左右方向如何定义

必须明确：横担的“左”和“右”不是 CloudCompare 视角里的左和右。  
它应该由杆塔局部坐标系决定。

推荐定义：

```text
沿线路方向为 along；
垂直线路方向为 side；
side 坐标小的一端为 left；
side 坐标大的一端为 right。
```

也就是：

```text
left  = local_side 最小的一侧
right = local_side 最大的一侧
```

这样同一份 LAS 不管在 CloudCompare 中怎么旋转视角，输出的左右端点都是稳定的。

如果后续业务要求“左/右”必须对应无人机飞行方向，则需要额外指定飞行方向：

```text
沿飞行方向看，左侧为 left，右侧为 right。
```

这时需要把 `along_axis_xy` 的正方向固定为飞行方向。

## 11. 过滤错误横担候选

横担推断容易受到误分绝缘子和误分导线影响，需要过滤。

建议过滤条件：

```text
该层绝缘子实例数量太少：删除；
该层附近导线点太少：删除；
横担左右长度太短：删除；
横担左右长度太长：删除；
高度低于杆塔高度一定比例：删除；
距离杆塔中心太远：删除。
```

推荐参数：

```text
min_insulators_per_crossarm = 1
min_line_points_near_crossarm = 30
min_crossarm_length = 1.0 米
max_crossarm_length = 30.0 米
min_height_ratio_on_tower = 0.35
```

示例：

```python
length = right_side - left_side

if length < min_crossarm_length:
    continue

if length > max_crossarm_length:
    continue

if line_points_near_layer < min_line_points_near_crossarm:
    continue
```

## 13. 伪代码

完整伪代码如下：

```python
读取 LAS
coord = [las.x, las.y, las.z]
cls = las.classification

tower_coord = coord[cls == 1]
line_coord = coord[cls == 2]
insulator_coord = coord[cls == 3]

towers = 对 tower_coord 做实例聚类和过滤
insulators = 对 insulator_coord 做实例聚类和过滤

for each tower in towers:
    tower.insulators = 找到属于该杆塔附近的绝缘子实例

    if tower.insulators 为空:
        continue

    along_axis_xy = 根据相邻杆塔中心或附近导线点估计线路方向
    side_axis_xy = [-along_y, along_x]

    layers = 按绝缘子中心 Z 高度分层

    for each layer in layers:
        layer_center_z = 该层绝缘子中心 Z 的中位数

        layer_insulator_points = 该层所有绝缘子点
        layer_line_points = 该高度附近、该杆塔附近的导线点

        if layer_line_points 太少:
            continue

        candidate_points = layer_insulator_points + layer_line_points

        local_side = (candidate_points.xy - tower.center.xy) dot side_axis_xy

        left_side = percentile(local_side, 2)
        right_side = percentile(local_side, 98)

        if right_side - left_side 长度不合理:
            continue

        left_xy = tower.center.xy + left_side * side_axis_xy
        right_xy = tower.center.xy + right_side * side_axis_xy

        crossarm_z = median(candidate_points.z)

        left_point_xyz = [left_xy.x, left_xy.y, crossarm_z]
        right_point_xyz = [right_xy.x, right_xy.y, crossarm_z]
        center_xyz = (left_point_xyz + right_point_xyz) / 2

        保存横担结果

按 tower_id 和高度从高到低排序横担
保存 JSON
```

## 14. 与绝缘子提取的关系

横担提取依赖绝缘子结果。  
建议先执行绝缘子实例提取，再执行横担提取。

推荐流程：

```text
分割 LAS
  -> 后处理清理杆塔、导线、绝缘子
  -> 提取绝缘子实例及 top/bottom/center
  -> 根据绝缘子和导线推断横担左右端点
```

如果绝缘子分割质量较差，横担推断也会变差。  
因此横担提取时要同时借用导线点做验证，不能只看绝缘子。

---

## 15. 注意事项

1. 横担没有显式类别，因此输出的是“推断位置”，不是直接分割出的横担点。
2. 左右端点必须基于杆塔局部坐标系定义，不能基于当前视角定义。
3. 横担方向通常垂直于线路走向，因此需要先估计 `along_axis_xy`。
4. 如果一个杆塔有多层绝缘子，应按高度分层，每层输出一个横担。
5. 如果某层只有导线但没有绝缘子，可靠性较低，建议不输出或标记为低置信度。
6. 如果某层绝缘子很多但附近没有导线，可能是误分，需要过滤。
7. 推荐使用百分位数计算左右端，避免单个噪声点把横担拉得过长。

## 16. 推荐落地版本

当前项目建议先实现这个版本：

```text
读取 clean LAS
+ 提取 tower / line / insulator 三类点
+ 杆塔实例聚类
+ 绝缘子实例聚类
+ 绝缘子绑定杆塔
+ 附近导线 PCA 估计线路方向
+ 按绝缘子高度分层
+ 每层用绝缘子点 + 导线点计算左右端
+ 输出 JSON
```

如果后续需要更稳定，可以继续升级：

```text
使用后处理脚本已生成的杆塔框和导线框；
用杆塔框确定中心；
用档距导线方向确定 along；
用绝缘子实例确定横担高度；
再输出左右端点。
```

这种升级版会比单纯从语义点聚类更稳定。
