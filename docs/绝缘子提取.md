# 绝缘子上下点和中心点提取逻辑

本文档用于说明：在模型已经完成输电线路四分类分割之后，如何从分割后的 `.las` 点云中提取 **每个绝缘子实例** 的最高点、最低点和中心点。

当前四分类语义约定：

```text
0：背景 / 地面
1：杆塔
2：导线
3：绝缘子
```

本文默认绝缘子类别为：

```text
insulator_class = 3
```

---

## 1. 输入和输出

### 输入

输入是推理或后处理后的 LAS 文件，例如：

```text
xxx_pred.las
xxx_clean.las
```

要求：

- LAS 中的 `classification` 字段已经保存语义分割结果。
- 绝缘子点的类别值为 `3`。
- 如果 LAS 中存在后处理追加的可视化点，例如 class `30/31`，应当忽略这些点。

### 输出

建议输出一个 JSON 文件，保存每个绝缘子的关键点：

```json
{
  "input": "xxx_clean.las",
  "insulator_class": 3,
  "coordinate_system": "las_global_xyz",
  "instances": [
    {
      "id": 1,
      "name": "绝缘子1",
      "point_count": 1250,
      "top_point_xyz": [526000.12, 3514000.35, 87.62],
      "bottom_point_xyz": [526000.20, 3514000.41, 82.15],
      "center_xyz": [526000.16, 3514000.38, 84.91],
      "center_point_xyz": [526000.15, 3514000.39, 84.90],
      "bbox_min_xyz": [525999.70, 3513999.95, 82.15],
      "bbox_max_xyz": [526000.62, 3514000.86, 87.62],
      "height": 5.47
    }
  ]
}
```

字段说明：

- `top_point_xyz`：该绝缘子实例中 `Z` 最大的真实点。
- `bottom_point_xyz`：该绝缘子实例中 `Z` 最小的真实点。
- `center_xyz`：该绝缘子实例所有点的几何中心，使用平均坐标计算，不一定是真实点。
- `center_point_xyz`：距离 `center_xyz` 最近的真实点。
- `bbox_min_xyz`：该绝缘子实例的轴对齐包围盒最小坐标。
- `bbox_max_xyz`：该绝缘子实例的轴对齐包围盒最大坐标。
- `height`：`top_point_z - bottom_point_z`。

注意：这里的坐标都使用 LAS 文件中的全局坐标，也就是 CloudCompare 右下角显示的坐标系。

---

## 2. 总体流程

整体处理流程如下：

```text
读取 LAS
  -> 取出 classification == 3 的绝缘子点
  -> 删除过小噪声点或离群点
  -> 对绝缘子点做实例聚类
  -> 每个聚类视为一个绝缘子实例
  -> 对每个实例提取最高点、最低点、中心点
  -> 按空间顺序给绝缘子编号
  -> 保存 JSON
```

---

## 3. 第一步：读取 LAS 坐标和类别

使用 `laspy` 读取 LAS：

```python
import laspy
import numpy as np

las = laspy.read(input_path)
coord = np.column_stack([las.x, las.y, las.z]).astype(np.float64)
cls = np.asarray(las.classification)
```

筛选绝缘子点：

```python
insulator_mask = cls == 3
insulator_coord = coord[insulator_mask]
insulator_indices = np.where(insulator_mask)[0]
```

如果没有绝缘子点：

```text
直接输出空 JSON：
instances = []
```

---

## 4. 第二步：绝缘子实例聚类

语义分割只告诉我们哪些点是绝缘子，但没有告诉我们“第几个绝缘子”。
所以需要把所有 class=3 的点进一步分成多个实例。

### 推荐方法：DBSCAN 聚类

DBSCAN 适合点云中的实例分离，因为它不需要提前指定绝缘子的数量。

逻辑：

```text
距离足够近的绝缘子点会被聚成一类；
距离太远的点会被分到不同实例；
孤立噪声点会被删除。
```

推荐参数：

```text
eps = 0.6 ~ 1.2 米
min_samples = 10 ~ 30
```

示例：

```python
from sklearn.cluster import DBSCAN

cluster = DBSCAN(
    eps=0.8,
    min_samples=15,
    n_jobs=-1,
).fit(insulator_coord)

labels = cluster.labels_
```

说明：

- `labels == -1` 表示噪声点，不作为绝缘子实例。
- `eps` 太小：一个绝缘子可能被拆成多个碎片。
- `eps` 太大：相邻绝缘子可能被合并成一个实例。

### 大点云优化方法：体素连通域

如果绝缘子点很多，DBSCAN 可能较慢，可以先做体素化，再做连通域。

逻辑：

```text
1. 按 voxel_size 把绝缘子点映射到体素网格。
2. 每个被占用的体素看作一个节点。
3. 26 邻域相连的体素归为同一个连通域。
4. 每个连通域对应一个绝缘子实例。
5. 再把体素实例编号映射回原始点。
```

推荐参数：

```text
voxel_size = 0.15 ~ 0.30 米
min_instance_points = 30 ~ 100
```

这种方法速度更快，更适合大规模 LAS 文件。

---

## 5. 第三步：过滤无效实例

聚类后需要删除明显不是绝缘子的噪声实例。

建议过滤条件：

```text
点数太少：删除；
高度太小：删除；
包围盒尺寸异常：删除；
距离杆塔和导线太远：可选删除。
```

推荐参数：

```text
min_instance_points = 30
min_height = 0.2 米
max_height = 20.0 米
max_extent_xy = 10.0 米
```

示例判断：

```python
points = insulator_coord[labels == instance_label]

bbox_min = points.min(axis=0)
bbox_max = points.max(axis=0)
extent = bbox_max - bbox_min
height = extent[2]

if len(points) < min_instance_points:
    continue

if height < min_height:
    continue

if height > max_height:
    continue

if max(extent[0], extent[1]) > max_extent_xy:
    continue
```

---

## 6. 第四步：提取最高点、最低点和中心点

对于每个绝缘子实例，设该实例点集为：

```python
points = insulator_coord[labels == instance_label]
```

### 最高点

最高点定义为 `Z` 最大的真实点：

```python
top_idx = np.argmax(points[:, 2])
top_point = points[top_idx]
```

### 最低点

最低点定义为 `Z` 最小的真实点：

```python
bottom_idx = np.argmin(points[:, 2])
bottom_point = points[bottom_idx]
```

这里的“最低点”就是高度方向最小的点，不是 `X/Y/Z` 三个坐标同时最小的点。

### 几何中心点

几何中心使用所有点的平均值：

```python
center = points.mean(axis=0)
```

这个中心点可能不是 LAS 中真实存在的点。

### 距离中心最近的真实点

如果后续飞行或可视化必须使用真实点，可以再找离中心最近的原始点：

```python
dist2 = np.sum((points - center[None, :]) ** 2, axis=1)
center_point = points[np.argmin(dist2)]
```

建议 JSON 中同时保存：

```text
center_xyz：数学中心
center_point_xyz：真实点中心
```

---

## 7. 第五步：绝缘子编号排序

为了让输出稳定，不能直接使用 DBSCAN 的原始 label。
建议按空间位置重新排序。

输电线路场景中，推荐排序方式：

```text
先按全局 X 从小到大；
如果 X 接近，再按全局 Y 从小到大；
如果 X/Y 接近，再按 Z 从高到低。
```

排序 key：

```python
sort_key = (
    center_xyz[0],
    center_xyz[1],
    -center_xyz[2],
)
```

排序后编号：

```text
绝缘子1
绝缘子2
绝缘子3
...
```

如果后续需要和杆塔绑定，也可以改成：

```text
先按杆塔编号；
再按绝缘子所在高度从上到下；
同一层内按局部 side 方向排序。
```

## 9. 推荐脚本参数

如果后续写成脚本，可以设计这些参数：

```bash
python tools/infer/extract_insulator_points.py \
  --input xxx_clean.las \
  --output xxx_insulators.json \
  --insulator-class 3 \
  --cluster-method dbscan \
  --eps 0.8 \
  --min-samples 15 \
  --min-instance-points 30 \
  --min-height 0.2 \
  --max-height 20.0 \
  --overwrite
```

参数说明：

- `--insulator-class`：绝缘子类别，默认 `3`。
- `--cluster-method`：聚类方式，可选 `dbscan` 或 `voxel_cc`。
- `--eps`：DBSCAN 的聚类半径。
- `--min-samples`：DBSCAN 的最少邻居点数。
- `--min-instance-points`：一个绝缘子实例最少点数。
- `--min-height`：一个绝缘子实例最小高度。
- `--max-height`：一个绝缘子实例最大高度。

## 10. 伪代码

完整伪代码如下：

```python
读取 LAS
coord = [las.x, las.y, las.z]
cls = las.classification

insulator_mask = cls == 3
insulator_coord = coord[insulator_mask]

if insulator_coord 为空:
    保存空 JSON
    结束

labels = 对 insulator_coord 做 DBSCAN 或体素连通域聚类

instances = []

for 每一个 label:
    if label == -1:
        continue

    points = insulator_coord[labels == label]

    if 点数太少:
        continue

    bbox_min = points.min(axis=0)
    bbox_max = points.max(axis=0)
    extent = bbox_max - bbox_min

    if 高度或尺寸不合理:
        continue

    top_point = points[np.argmax(points[:, 2])]
    bottom_point = points[np.argmin(points[:, 2])]
    center = points.mean(axis=0)
    center_point = points[距离 center 最近的真实点]

    instances.append({
        "point_count": len(points),
        "top_point_xyz": top_point,
        "bottom_point_xyz": bottom_point,
        "center_xyz": center,
        "center_point_xyz": center_point,
        "height": top_point[2] - bottom_point[2],
        "bbox_min_xyz": bbox_min,
        "bbox_max_xyz": bbox_max,
    })

instances 按 center 的 X、Y、Z 排序

重新编号：
    id = 1, 2, 3...
    name = 绝缘子1, 绝缘子2, 绝缘子3...

保存 JSON
```

## 11. 和杆塔、导线的关系过滤

如果分割结果中有背景误分为绝缘子，可以增加关系约束。

绝缘子通常满足：

```text
靠近杆塔；
靠近导线；
位于杆塔上部或横担附近；
不会孤立出现在纯背景区域。
```

可选过滤逻辑：

```text
1. 对每个绝缘子实例，计算它到最近杆塔点的距离。
2. 计算它到最近导线点的距离。
3. 如果同时远离杆塔和导线，则删除。
```

推荐阈值：

```text
max_distance_to_tower = 5.0 ~ 8.0 米
max_distance_to_line = 1.0 ~ 2.0 米
```

这个逻辑可以减少背景中误分出来的小黄色点。

---

## 12. 注意事项

1. “最高点”和“最低点”应该按 `Z` 高度判断。
2. 中心点建议同时保存数学中心和最近真实点。
3. 坐标必须使用 LAS 全局坐标，不要使用推理时内部 `coord -= coord.min()` 后的相对坐标。
4. 如果输入 LAS 是经过画框脚本处理后的文件，需要忽略 class `30/31` 的可视化点。
5. 聚类参数要根据点云密度调整。点云密度越稀疏，`eps` 需要适当变大。
6. 如果一个绝缘子串被分成多个碎片，说明 `eps` 太小或 `min_samples` 太大。
7. 如果多个绝缘子被合成一个实例，说明 `eps` 太大。

---

## 13. 推荐落地版本

当前项目中建议优先实现：

```text
laspy 读取 LAS
+ classification == 3 筛选绝缘子点
+ DBSCAN 聚类
+ 点数/高度过滤
+ 提取 top/bottom/center
+ 保存简洁 JSON
```

后续如果数据量很大或 DBSCAN 速度慢，再切换为：

```text
体素连通域聚类
```

如果用于仿线飞行，建议最终使用：

```text
top_point_xyz
bottom_point_xyz
center_point_xyz
```

其中 `center_point_xyz` 是真实点，更适合作为飞行任务中的目标点或可视化标记点。
