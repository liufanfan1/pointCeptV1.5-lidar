# `postprocess_tower_line_clean.py` 后处理逻辑

## 1. 脚本用途

`tools/infer/postprocess_tower_line_clean.py` 是输电线路点云后处理流程的第一阶段，输入模型推理得到的四分类 LAS/LAZ，输出清理后的四分类 LAS/LAZ 和调试报告 JSON。

分类约定如下：

| classification | 类别 |
|---:|---|
| `0` | 背景、地面、植被等 |
| `1` | 杆塔 |
| `2` | 导线 |
| `3` | 绝缘子 |
| `30` | 画框阶段生成的拟合导线点 |
| `31` | 画框阶段生成的框线、文字等可视化点 |

本脚本负责：

- 清理贴近地面的导线和绝缘子误分点；
- 对杆塔点进行聚类、过滤、合并和塔脚恢复；
- 根据杆塔与导线、绝缘子的空间关系删除假杆塔；
- 清理真实杆塔附近脱离主体的组件和塔底外围散点；
- 输出清理后的 LAS 和清理报告 JSON。

本脚本不负责生成杆塔框、档距框、拟合导线或导线编号。画框应在清理完成后使用：

```text
tools/box_lineAndTower/draw_tower_line_boxes.py
```

## 2. 总体处理顺序

脚本按以下顺序执行：

```text
读取 LAS
  -> 删除旧 class 30/31 合成点
  -> 按四分类重新着色
  -> 清理近地导线/绝缘子
  -> 杆塔体素聚类
  -> 杆塔组件基础过滤
  -> 合并物理杆塔
  -> 恢复杆塔底部组件
  -> 杆塔与导线/绝缘子关系过滤
  -> 清理杆塔附近脱离组件和塔底散点
  -> 将无效杆塔点改为背景
  -> 再次按类别着色
  -> 写出清理 LAS 和报告 JSON
```

## 3. 读取输入与清理旧可视化点

脚本读取 LAS/LAZ 后会检查是否存在 `classification` 维度。若不存在，直接报错。

默认会物理删除输入中已有的 `class=30` 和 `class=31` 合成点，防止旧拟合线、旧框线和旧文字影响统计或重复显示。

```bash
--keep-existing-synthetic-points
```

传入上面的参数可以保留旧合成点，但正常清理流程不建议使用。

这里需要区分两类“删除”：

- 旧 `class=30/31` 点：从 LAS 点记录中物理删除；
- 误分的 `class=1/2/3` 原始点：保留坐标，只把 `classification` 改成 `0`。

## 4. 清理接近地面的导线和绝缘子

该逻辑默认启用，并且在杆塔关系判断之前运行，避免贴地的假导线和假绝缘子错误地帮助假杆塔通过过滤。

处理对象为：

```text
class=2 导线点
class=3 绝缘子点
```

地面高度估计流程如下：

1. 只使用 `class=0` 背景点估计地面；
2. 将背景点按 XY 平面划分为默认 `2.0m x 2.0m` 网格；
3. 每个网格使用背景点 Z 值的第 `5%` 分位数作为局部地面高度；
4. 如果导线或绝缘子所在网格没有背景点，则使用全部背景点 Z 值的第 `2%` 分位数作为兜底地面高度；
5. 若候选点满足下面的条件，则将其类别改为背景：

```text
点的 Z <= 局部地面 Z + 2.0m
```

主要参数如下：

| 参数 | 默认值 | 含义 |
|---|---:|---|
| `--low-line-insulator-height` | `2.0` | 距局部地面不高于该值的导线/绝缘子改为背景，单位 m |
| `--low-line-insulator-ground-grid-size` | `2.0` | 局部地面网格边长，单位 m |
| `--low-line-insulator-ground-percentile` | `5.0` | 每个网格估计地面时使用的 Z 分位数 |
| `--low-line-insulator-global-ground-percentile` | `2.0` | 网格无背景点时使用的全局 Z 分位数 |
| `--no-remove-low-line-insulator` | 关闭开关 | 完全关闭该清理逻辑 |

如果真实导线弧垂很低并被误删，应减小 `--low-line-insulator-height`；如果贴地误分仍较多，可以适当增大该值。

## 5. 杆塔体素聚类

脚本只对 `class=1` 杆塔点进行聚类：

1. 按默认 `0.50m` 体素大小离散化；
2. 对非空体素建立邻接关系；
3. 默认使用 26 邻域计算三维连通域；
4. 将每个连通域还原为一个候选杆塔组件；
5. 统计每个组件的原始点数、包围盒、尺寸和中心。

主要参数：

| 参数 | 默认值 | 含义 |
|---|---:|---|
| `--tower-voxel-size` | `0.50` | 杆塔连通域聚类体素大小，单位 m |
| `--tower-connectivity` | `26` | 体素连通方式，可选 `6/18/26` |

体素越小，组件划分越细，但计算量和碎片数量会增加；体素越大，附近但实际分离的物体更容易被连接。

## 6. 候选杆塔组件基础过滤

每个杆塔组件依次检查以下条件，前面已经失败的条件会成为该组件的主要删除原因：

| 检查项 | 默认条件 | 删除原因字段 |
|---|---:|---|
| 原始点数 | 不少于 `200` | `too_few_points` |
| 杆塔高度 | 不低于 `6.0m` | `too_low` |
| XY 水平尺寸 | X/Y 最大尺寸不小于 `0.30m` | `xy_too_small` |
| 附近导线 | 中心 XY 半径 `18m` 内至少有导线 | `no_nearby_line` |
| 附近导线数量 | 至少 `100` 个导线点 | `too_few_nearby_line_points` |
| 高出局部地面的高度 | 杆塔顶高出地面至少 `4.0m` | `too_low_above_ground` |

杆塔局部地面估计使用组件中心 XY 周围默认 `8m` 内的背景点，并取这些背景点 Z 值的第 `5%` 分位数；附近没有背景点时，使用全部背景点 Z 值的第 `2%` 分位数。

可选参数 `--require-line-through-tower` 会额外检查杆塔上部区域是否有足够导线点，但默认不启用。

## 7. 合并同一座物理杆塔

通过基础过滤的组件会根据 XY 中心距离合并为物理杆塔。默认合并半径为 `10m`：

```bash
--merge-tower-xy-radius 10.0
```

合并使用连通关系，因此具有传递性。例如 A 靠近 B、B 靠近 C 时，A、B、C 可能合并为同一组，即使 A 与 C 的距离略大于阈值。

## 8. 恢复杆塔底部组件

严格的点数、高度和关系过滤可能把塔脚附近的小组件删除。脚本默认执行塔脚恢复，只恢复原本就是 `class=1` 的组件，不会把背景点凭空改成杆塔。

被删除组件满足以下条件时可以恢复：

- 原始点数不少于 `20`；
- 组件 XY 中心位于有效杆塔包围盒向外扩 `3m` 的范围内；
- 组件最高点不高于杆塔底部加 `8m`。

相关参数：

| 参数 | 默认值 |
|---|---:|
| `--recover-base-min-points` | `20` |
| `--recover-base-xy-margin` | `3.0` |
| `--recover-base-z-margin` | `8.0` |
| `--no-recover-tower-base` | 关闭塔脚恢复 |

## 9. 杆塔空间关系过滤

组件合并并恢复塔脚后，脚本会临时计算杆塔包围框，用于关系判断。这个框只在内存中参与计算，本脚本不会把框写入 LAS。

### 9.1 默认启用的关系过滤

默认启用以下两项严格过滤：

1. 杆塔上部与连续导线三维接触；
2. 杆塔顶部附近存在足够导线点。

“导线接触”不是简单统计杆塔上方的所有蓝色点。脚本先对导线做 `0.50m` 三维体素连通分析，只保留点数不少于 `100`、XY 长度不少于 `30m` 的连续导线组件，再执行接触判断。这样可以减少短小导线误分对结果的干扰。

接触判断默认只检查杆塔高度上部 `45%` 以上的杆塔点。若至少 `20` 个杆塔点到连续导线的三维最近距离不大于 `1.5m`，则认为杆塔与线路接触。

杆塔顶部导线判断会在杆塔 XY 包围范围向外扩 `6m`，高度范围为杆塔顶部向下 `5m`、向上 `8m`，默认至少需要 `30` 个导线点。

### 9.2 可选关系过滤

以下逻辑已实现，但只有显式传入对应开关时才启用：

| 开关 | 判断内容 |
|---|---|
| `--require-line-through-tower` | 导线是否从杆塔上部区域穿过，并延伸到杆塔两侧 |
| `--require-line-inside-tower` | 杆塔中上部核心区域内是否有足够导线点 |
| `--require-continuous-line-inside-tower` | 导线是否连续进入杆塔核心区域，而不是零散或擦边 |
| `--require-side-line-near-tower` | 杆塔框左侧或右侧是否存在导线，排除仅从上方经过的线路 |
| `--require-insulator-near-tower` | 杆塔上部附近是否存在足够绝缘子点 |
| `--require-insulator-line-bridge` | 是否有足够绝缘子点同时靠近杆塔和导线 |
| `--remove-bare-pole-towers` | 删除上部没有横向展开结构的光杆或瘦杆误检 |
| `--require-connected-line-span` | 杆塔是否通过有效档距连接左侧或右侧相邻杆塔 |

这些严格条件不建议一次全部打开。应根据数据中的误检类型逐项启用，否则可能把点云稀疏或绝缘子漏分的真实杆塔删除。

## 10. 清理真实杆塔附近的零散误检

完成杆塔关系过滤后，脚本还会做两层精细清理。

第一层是组件级清理：同一物理杆塔中，以高度最高、点数最多的组件作为主体。其他组件与主体的 XY 包围盒间距超过默认 `3m` 时，视为脱离主体的误检组件。

第二层是点级清理：

1. 使用杆塔高度 `35%~75%` 范围内点的 XY 中位数估计杆塔主体中心；
2. 只检查杆塔底部 `35%` 高度范围内的点；
3. 底部点到主体中心的 XY 距离大于 `6m` 时，标记为外围散点并改为背景。

相关参数：

| 参数 | 默认值 |
|---|---:|
| `--tower-nearby-component-max-gap` | `3.0` |
| `--tower-clutter-lower-height-ratio` | `0.35` |
| `--tower-clutter-base-radius` | `6.0` |
| `--no-prune-nearby-tower-clutter` | 关闭上述两层精细清理 |

## 11. 分类回写与颜色

所有未通过检查的杆塔点以及近地导线/绝缘子点都不会从原始点云中物理删除，而是执行：

```text
classification: 原类别 -> 0（背景）
```

如果 LAS 包含 RGB，脚本默认按最终分类重新着色：

| 类别 | 显示颜色 |
|---|---|
| 背景 | 灰色 |
| 杆塔 | 红色 |
| 导线 | 蓝色 |
| 绝缘子 | 黄色 |

使用 `--no-recolor` 可以只修改分类而不覆盖 RGB。

## 12. 输出文件与报告内容

脚本输出两个文件：

1. `--output` 指定的清理后 LAS/LAZ；
2. `--report` 指定的 JSON。未指定时，默认保存为 `<output文件名>_clean_report.json`。

报告主要包含：

- 清理前和清理后的分类点数；
- 杆塔组件总数、保留数和删除数；
- 最终物理杆塔数量及杆塔属性；
- 恢复的塔脚组件数量；
- 各种关系检查删除的杆塔数量；
- 删除的脱离组件和塔底散点数量；
- 近地导线、近地绝缘子各自改为背景的点数；
- 用于地面估计的网格数量；
- 旧 `class=30/31` 合成点的物理删除数量；
- 总耗时、杆塔清理耗时和本次运行的全部参数。

## 13. 推荐执行命令

项目默认使用 `transmission-line` 预设，正常情况下只需要提供输入、输出和覆盖开关：

```bash
python tools/infer/postprocess_tower_line_clean.py \
  --input /path/to/prediction.las \
  --output /path/to/prediction_clean.las \
  --overwrite
```

指定报告路径：

```bash
python tools/infer/postprocess_tower_line_clean.py \
  --input /path/to/prediction.las \
  --output /path/to/prediction_clean.las \
  --report /path/to/prediction_clean_report.json \
  --overwrite
```

近地误分较多时，可以提高近地清理高度：

```bash
--low-line-insulator-height 3.0
```

真实低垂导线被误清理时，可以减小阈值：

```bash
--low-line-insulator-height 1.0
```

需要复现较宽松的旧流程时使用：

```bash
--preset legacy
```

`legacy` 会放宽杆塔组件阈值并关闭部分默认关系检查，仅建议用于旧实验对比，不建议作为当前输电线路生产流程的默认配置。

## 14. 使用注意事项

- 输入应当是模型原始四分类结果，不建议把已经追加框线和拟合线的 LAS 反复作为清理输入；
- 局部地面估计依赖 `class=0` 背景点，若背景分类严重缺失，近地清理结果会退化为全局地面高度判断；
- `--low-line-insulator-height` 过大可能误删真实低垂导线，过小则可能保留地面附近误分；
- `--merge-tower-xy-radius` 过大可能把相邻假目标并入真实杆塔；
- 接触、绝缘子和有效档距条件越严格，对模型漏分越敏感；
- 清理完成后，再将输出 LAS 交给画框脚本，能够避免无效杆塔进入杆塔框和档距框生成流程。
